# 他システム連携における課題：セルコメントとセル位置の分離問題

---

## 背景

iReporter Excel Add-in は、Excelセルのコメント（行0〜15）にフィールドのメタデータを格納する。  
このメタデータを使って別システムと連携することを検討する際、以下の構造的な課題が存在する。

---

## 課題：コメント内にセル番号が含まれていない

### 現状の構造

```
ClusterInfo（Add-in内部オブジェクト）
├─ Row, Col       ← Excelのセルオブジェクトから取得（コメント外）
└─ _texts[]       ← コメントテキスト（行0〜15）
     ├─ 行0: クラスタ名
     ├─ 行1: クラスタ種別キー
     ├─ 行2: クラスタインデックス
     └─ ...（セル位置情報は存在しない）
```

**コメントテキスト（行0〜15）は「このフィールドは何か」を定義するが、「どこにあるか」は持っていない。**

セル位置（行番号・列番号）は、Excelファイルを開いてセルオブジェクトの `.Row` / `.Column` を参照することでのみ取得できる。

### 影響

| シナリオ | セル位置の取得可否 |
|---------|:---------------:|
| Add-inがExcelを開いた状態でコメントを読む | ✓ 取得できる |
| コメントテキストだけを受け取った場合 | ✗ 取得できない |
| Excelを経由せずメタデータだけをやりとりする場合 | ✗ 取得できない |

---

## 連携設計の選択肢と比較

### A案：Excelを都度読んでマッピングを生成する

```
他システム / 連携ツール
        │
        ▼
  Excelファイルを開く
        │
        ▼
  全シートのコメント付きセルを走査
  （現在の InitClusterList() と同じアプローチ）
        │
        ▼
  {
    sheetName: "Sheet1",
    cell: "B3",          ← セルから取得
    clusterName: "氏名",
    typeKey: "KeyboardText",
    clusterIndex: 0
  }
```

**メリット**
- 既存の仕様を変更しない
- Excelが正として機能する（単一の情報源）

**デメリット**
- 連携のたびにExcelを開く必要がある
- Excel非依存のシステムには組み込みが難しい
- セル位置がシート上で変わった場合に自動追従できない

---

### B案：外部カタログにセル番号を持たせる

コメントの内容とセル位置を**外部（JSONやDB）で一元管理**する設計。

```json
[
  {
    "sheetName": "Sheet1",
    "cell": "B3",
    "clusterName": "氏名",
    "typeKey": "KeyboardText",
    "clusterIndex": 0,
    "readOnly": false,
    "extension": false
  },
  {
    "sheetName": "Sheet1",
    "cell": "B5",
    "clusterName": "報告日",
    "typeKey": "CalendarDate",
    "clusterIndex": 0
  }
]
```

**メリット**
- Excelを開かずにセル位置とメタデータを把握できる
- 連携システムがExcelに非依存で設計できる

**デメリット**
- カタログとExcel実態が乖離するリスクがある（二重管理）
- カタログの更新管理フローが必要になる
- セル位置がExcel側で変更されたときにカタログとの同期が必要

---

### C案：コメントにセル番号を自己記述させる（仕様変更）

コメントの行15以内（または16以降の未使用行）にセル自身のアドレスを追記する設計変更。

```
氏名
KeyboardText
0
0
0
（入力パラメータ）
B3          ← 備考1などにセルアドレスを入れる（仕様変更）
...
```

**メリット**
- コメントだけで自己完結する

**デメリット**
- セルを移動したときにコメントの記述が古くなる（自動追従しない）
- 既存の備考フィールドを流用すると、他の用途と競合するリスクがある
- iReporter本体がこの値をどう解釈するか未確認（副作用の懸念あり）
- **現状では非推奨**

---

## 推奨アプローチ

### 短期（デバッグ・PoC）

**A案を採用する。**  
ExcelファイルをAdd-inと同じ方法で走査し、セル位置+コメントのマッピングを生成してから連携テストに使う。

### 中長期（本番連携）

**B案（外部カタログ）を採用し、以下の運用ルールを設ける。**

1. カタログはExcelを正として自動生成する（手動管理しない）
2. Excelのコメントが更新されたタイミングでカタログを再生成する
3. カタログには `lastGeneratedAt`（生成日時）を付与し、乖離を検知できるようにする
