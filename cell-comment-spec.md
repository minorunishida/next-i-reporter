## 概要

このAdd-inは、Excelシート上の各セルに**iReporter帳票定義のメタデータ**をコメントとして埋め込む。  
セルコメントはiReporterがExcelを読み込む際に**フィールドマッピング情報**として解釈される。

```
Excel セル
 └─ セルコメント（改行区切りの16行以上のテキスト）
      ├─ 行0〜15  : クラスタ（フィールド）定義
      └─ 行16〜24 : テーブルセル専用の追加情報（テーブルに属する場合のみ）
```

---

## コメント構造の基本

### フォーマット

```
<クラスタ名>\n
<クラスタ種別キー>\n
<クラスタインデックス>\n
<書き込み可フラグ>\n
<拡張フラグ>\n
<入力パラメータ>\n
<備考1>\n
<備考2>\n
<備考3>\n
<備考4>\n
<備考5>\n
<備考6>\n
<備考7>\n
<備考8>\n
<備考9>\n
<備考10>
```

### 区切り文字

| 項目 | 仕様 |
|------|------|
| 行区切り | `\n`（LF のみ） |
| `\r` の扱い | 読み込み時に除去される（`SplitLine` 参照） |
| 行数 | 最低16行。行が足りない場合はシステムが自動補完する |
| 最大行数 | テーブルなしの場合は行0〜15（16行）。テーブルありは行0〜24（25行） |

### 自動補完ルール（参照：`ThisAddIn.cs` WriteComment メソッド）

コメントが16行未満の場合、不足行を以下のルールで自動補完する：

```csharp
while (commentList.Count < 16)
    commentList.Add(commentList.Count == 3 || commentList.Count == 4 ? "0" : "");
```

| 行番号 | 補完値 |
|--------|--------|
| 行3（書き込み可フラグ） | `"0"` |
| 行4（拡張フラグ）       | `"0"` |
| その他                  | `""` （空文字） |

> **実装担当メモ**：手動でコメントを書く際は必ず16行分を明示的に記述すること。  
> 補完ロジックに依存すると、書き込み時にAdd-inを経由しなければ正しく機能しない場合がある。

### 行0：クラスタ名

iReporter上でフィールドに表示されるラベル文字列。  
Add-inの自動判定では、選択セル周辺のセルテキストから自動生成される。

- 空文字でも動作するが、連携先システムがこの値を参照する場合は要確認
- `CaptionPriority` 設定（上・左・絶対上・絶対左）に基づき自動生成される

---

### 行1：クラスタ種別キー

**最重要フィールド。** iReporterのフィールド種別を決定する。  
有効な値は[クラスタ種別キー一覧](#クラスタ種別キー一覧)を参照。

**デバイスによる自動変換ルール**（`ApplicationConfig.xml` の `autoConvert` 属性）：

| 指定キー | Windowsでの変換先 |
|--------|---------------|
| `Handwriting` | `KeyboardText` |
| `Date` | `CalendarDate` |
| `CodeReader` | `QRCode` |

> **レビュー指摘**：システム連携でWindowsとiOS両方を対象にする場合、  
> 自動変換されるキーを直接記述しないこと。変換先を明示的に使用することを推奨する。  
> 例：iOSターゲットでも `Date` より `CalendarDate` を使用する方が安全。

---

### 行2：クラスタインデックス

同一シート内で同じ種別キーのフィールドが複数ある場合の連番（0始まり）。

```
例：KeyboardText フィールドが3つある場合
  1つ目：行2 = "0"
  2つ目：行2 = "1"
  3つ目：行2 = "2"
```

- 空文字 `""` でも動作するが、連携システムがこの値を参照する場合は必須
- 数値文字列のみ有効（整数）。負の値は「未設定」と解釈される

---

### 行3：書き込み可フラグ

> ⚠️ **命名に注意**：変数名は `readOnlyCheckBox` だが、チェックが入った状態が「書き込み可」を意味する。  
> 値の意味は直感と逆になっている（コード：`readOnlyCheckBox.Checked ? "0" : "1"`）。

| 値 | 意味 |
|:--:|------|
| `"0"` | **書き込み可**（デフォルト） |
| `"1"` | **読み取り専用** |

**重要な制約**：このフラグは**拡張フラグ（行4）が `"1"` の場合のみ有効**。  
拡張フラグが `"0"` の場合、このフラグは無視され `"0"` として扱われる。

```
行4 = "0"（拡張なし） → 行3 は常に "0" として扱われる
行4 = "1"（拡張あり） → 行3 の値が有効
```

---

### 行4：拡張フラグ

| 値 | 意味 |
|:--:|------|
| `"0"` | 拡張機能なし（デフォルト） |
| `"1"` | 拡張機能あり（行3の書き込み可フラグが有効になる） |

> **実装担当メモ**：デバッグ用の手動コメントでは通常 `"0"` を設定する。  
> `"1"` を使う場合は行3の設定も必ずセットで管理すること。

---

### 行5：入力パラメータ

クラスタ種別（行1）が `Action` 等アクション系の場合に使用する追加パラメータ。  
UIでは読み取り専用として表示される（`inputParameterTextBox`）。

- 種別キーを変更した際にAdd-inは自動的にこのフィールドをクリアする
- 手動でコメントを書く場合は、種別に対応した正しいパラメータを記述すること
- 不明な場合は空文字 `""` とする

---

### 行6〜15：備考1〜10

自由記述フィールド。連携システム独自の情報を格納可能。  
Add-in本体はこれらの値を解釈しない。

## テーブルセルの拡張フォーマット

テーブルに属するセルは、通常の16行コメント（行0〜15）に加えて、  
**行16〜24** にテーブル固有情報が追加される。

### テーブル拡張行の仕様

| 行# | フィールド名 | 型 | 説明 |
|----:|------------|:--:|------|
| **16** | テーブルNo | 数値文字列 | 同一テーブルに属するセルが共有する識別番号（1以上） |
| **17** | テーブル名 | 文字列 | テーブルの表示名 |
| **18** | 連携出力フラグ | `"0"` / `"1"` | `"1"` = コラボレーション出力あり |
| **19** | 列インデックス | 数値文字列 | 列の番号（1始まり）。**読み取り専用**（Add-inが自動設定） |
| **20** | 列名 | 文字列 | 列の表示名 |
| **21** | 列キー | 文字列（最大5文字推奨） | 列の識別キー。未設定の場合は `F_001` のような自動生成値 |
| **22** | 列型 | 文字列（列挙） | 列のデータ型（後述） |
| **23** | 行インデックス | 数値文字列 | 行の番号（1始まり）。**読み取り専用**（Add-inが自動設定） |
| **24** | 行名 | 文字列 | 行の表示名 |

### 列型（ColType）の値

| 値 | 意味 |
|:--:|------|
| `numeric` | 数値 |
| `date` | 日付 |
| `interval` | 時間間隔 |
| `text` | テキスト |
| `calculate` | 計算式 |
| `NONE` | 出力なし |
| `INVALID` | 使用不可（ソフトウェアが自動設定、手動設定は非推奨） |

> **実装担当メモ**：列インデックス（行19）と行インデックス（行23）はAdd-inの `Table.Adjust()` が  
> 自動計算して書き込む値であり、手動で書いた値は**上書きされる**。  
> テーブル構造は同じ `TableNo` を持つセルの集合から自動的に再構築されるため、  
> `TableNo`・`TableName`・`ColKey`・`RowName` に集中して正確に記述すること。

## バリデーションルールと注意事項

### テーブルNoの扱い

```csharp
// TableItem.cs の読み込み条件
int.TryParse(item.TableNo, out var n);
if (n < 1) continue;  // TableNo が 1 未満のセルはテーブルと認識されない
```

- **TableNo は 1 以上の整数を使用すること**（0 や負の値はテーブルとして認識されない）
- 同一テーブルに属するすべてのセルは **同一の TableNo** を持たなければならない
- 同一テーブルのセルは **必ず同一シート内**に存在すること

### クラスタインデックスの重複

- 同一シート・同一種別キーで同じインデックス値が複数存在しないようにすること
- Add-inは重複チェックを行わないため、連携先システムが誤動作する可能性がある

### 最大選択セル数制限

```csharp
const int MAX = 1000;
```

Add-inの一括書き込みは1000セルまで。システム連携でコメントを生成する場合も同様の上限を考慮すること。

### 改行コードの注意

- コメント内の行区切りは `\n`（LF）のみ
- `\r`（CR）は読み込み時に除去されるが、**書き込み時に `\r\n` を使用するとフィールドが正しく分割されない可能性がある**
- システム連携でコメントを生成する場合は **LF のみ**を使用すること

### tableColType が `invalid` のクラスタ種別

以下の種別は**テーブルセルとして使用できない**。これらを持つセルに行16以降のテーブル情報を付加しても、テーブル整合性チェックでエラーになる：

`FixedText`, `FreeText`, `FreeDraw`, `Image`, `DrawingImage`, `AudioRecording`, `Scandit`, `EdgeOCR`

### 結合セルの扱い

- 結合セルには**結合範囲の左上セル**のみにコメントを設定すること
- Add-inは結合セルを検出し、左上セル以外をスキップする処理を内包している

### 🟡 注意：書き込み可フラグの命名の罠

コード中の変数名は `readOnlyCheckBox` だが、値の意味は：

```
"0" = 書き込み可（チェックあり）
"1" = 読み取り専用（チェックなし）
```

直感と逆のため、システム連携コードを書く際は**混乱しやすい箇所**。  
コードレビューで必ずチェックすること。

### 🟡 注意：AI判定エンドポイントへの依存
